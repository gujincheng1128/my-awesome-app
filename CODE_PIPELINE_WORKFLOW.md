### 目标
- 理解用户真实需求
- 明确项目范围和边界
- 识别关键干系人
- 定义成功标准

### 技能组合
| 技能 | 用途 | 触发时机 |
|------|------|----------|

| `deer-data-analysis` | 数据需求评估 | 涉及数据处理 |

### 工作流

```
用户输入: "帮我做一个用户管理系统"

         ▼

```

```
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 1. 写最简单的失败测试                                        ││
│  │ 2. 运行测试 → 确认失败                                       ││
│  │ 3. 编译错误是可接受的                                       ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  🟢 GREEN - 让测试通过                                           │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 1. 写最小代码使测试通过                                     ││
│  │ 2. 不要优化，只求通过                                       ││
│  │ 3. 运行测试 → 确认通过                                     ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  🔵 REFACTOR - 重构优化                                          │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 1. 提取常量                                                 ││
│  │ 2. 消除重复                                                 ││
│  │ 3. 重命名提高可读性                                         ││
│  │ 4. 运行测试 → 确认全部通过                                  ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```
